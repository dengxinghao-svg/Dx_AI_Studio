# 工作流执行层验收清单

这份清单按“管理页面 -> 无限画布 -> 工作流执行 -> 文本节点 -> 项目恢复”的顺序来验收。

## 一、管理页面

1. 打开根目录 `index.html`
2. 确认能自动进入管理页面
3. 点击“新建项目”
4. 确认能进入无限画布编辑页
5. 返回管理页面后，确认刚创建的项目排在最新位置

验收通过标准：

- 管理页面能正常打开
- 能创建项目并进入编辑器

## 二、无限画布基础能力

1. 在左侧快速创建 `text / image / video / asset` 节点
2. 拖动节点
3. 框选多个节点
4. 用端口拖拽创建连线
5. hover 连线并用剪刀断开
6. 选中多个节点后打组，再解组

验收通过标准：

- 节点增删改拖动正常
- 连线可创建、可删除
- 分组和解组正常

## 三、工作流执行层

### 1. 执行当前节点

1. 选中一个节点
2. 点击右侧“执行当前节点”
3. 观察节点状态：
   - `queued`
   - `running`
   - `done`
4. 观察右侧“工作流执行”卡片：
   - 执行模式
   - 最近运行
   - 运行次数
   - 输入数
   - 输出类型
   - 执行摘要

通过标准：

- 节点状态会变化
- 右侧卡片会更新

### 2. 执行整条链

1. 创建一条多节点链路，例如：
   - text -> image -> video
2. 选中链路中的起始节点或中间节点
3. 点击“执行整条链”
4. 观察整条链上节点依次进入：
   - `queued`
   - `running`
   - `done`

通过标准：

- 链上节点按顺序执行
- 下游节点会读取上游输入

### 3. 重置状态

1. 执行一轮节点或链路
2. 点击“重置状态”
3. 检查选中链路或当前节点状态是否回到默认值

通过标准：

- `status / output / error / runCount / lastRunAt` 被清空或恢复默认

## 四、右侧工作台

1. 选中一个未执行节点
2. 看“工作流执行”卡片默认内容
3. 执行后再看卡片变化
4. 再选中另一个节点，确认内容切换

通过标准：

- 右侧信息会随节点切换
- 未执行节点和已执行节点显示不同内容

## 五、文本节点

1. 创建文本节点
2. 在节点内输入富文本内容
3. 使用顶部工具条：
   - H1
   - H2
   - H3
   - 引用
   - 加粗
   - 斜体
   - 列表
   - 编号列表
   - 分隔线
4. 点击全屏按钮进入文档模式
5. 在全屏中编辑后关闭

通过标准：

- 文本节点内容会同步
- 节点内和全屏内容是一致的
- 关闭全屏后内容不丢失

## 六、项目保存与恢复

1. 在画布中建立一组节点和连线
2. 执行一次节点或链路
3. 返回管理页面
4. 再次打开项目
5. 检查：
   - 节点是否还在
   - 连线是否还在
   - 文本内容是否还在
   - 工作流字段是否恢复

通过标准：

- 快照恢复后，节点结构和运行信息仍然存在

## 七、快捷方式回归

1. `Ctrl + 鼠标滚轮` 缩放画布
2. `Ctrl + +` / `Ctrl + -` 缩放画布
3. `Shift + 左键` 多选节点
4. `Del` 删除节点
5. `Ctrl + C / X / V` 复制剪切粘贴节点
6. `Esc` 退出文本全屏

通过标准：

- 快捷键不与新执行层冲突

## 八、已知限制

当前若以下情况出现，不算本轮验收失败：

1. 节点执行结果仍然是 mock 输出，不是真实模型返回
2. 后端接口仍是骨架占位
3. 没有接入完整自动化测试
